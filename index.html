핵심 개념

Software 3.0이란?

  • 1.0: 인간이 코드를 직접 작성 (C, Java, Python...)
  • 2.0: 데이터로 모델을 학습시킴 (머신러닝)
  • 3.0: 컨텍스트 윈도우가 새로운 코드 — 자연어로 의도를 전달하면 AI가 실행

"이제 PM은 코드를 짜는 사람이 아니라, 에이전트의 시스템을 설계하는 사람입니다."

동작 원리

에이전트는 어떻게 일할까?

🧠 인지 (Perceive)

코드, 파일, 에러 메시지 등 현재 상태를 파악

📋 계획 (Plan)

어떤 순서로 무엇을 수정할지 전략 수립

⚡ 행동 (Act)

코드 수정, 테스트 실행, 파일 생성 등 실행

👁️ 관찰 (Observe)

결과를 확인하고, 오류가 있으면 다시 루프

이 루프를 인간 개입 없이 수차례 반복 → "에이전틱 루프"

핵심 개념

하네스(Harness) 엔지니어링

에이전트를 둘러싼 외부 장치를 설계하는 것.
말(馬)에게 고삐를 씌우듯, AI에게 구조적 제어판을 만들어줍니다.

  • 어떤 도구(Tools)를 줄지 결정
  • 어떤 훅(Hooks)으로 위험 행동을 차단할지 설계
  • 어떤 MCP(외부 시스템)에 연결할지 선택
  • 어떤 스킬(Skills)을 패키징하여 재사용할지 구성

아키텍처

에이전트의 5계층 구조

1 메모리 CLAUDE.md — 에이전트의 "헌법". 규칙과 관습을 항상 기억
2 지식 Skills — 특정 과업의 전문 지식을 담은 문서
3 가드레일 Hooks — 위험한 행동을 자동으로 차단/제어
4 위임 Subagents — 전문 하위 에이전트에게 작업 분산
5 배포 Plugins — 검증된 스킬을 팀 전체에 표준화 배포

가드레일

훅(Hook)으로 에이전트 통제하기

PreToolUse 훅

행동 전에 실행. 위험 명령어 차단
예: rm -rf 실행 전 무조건 인간에게 확인

PostToolUse 훅

행동 후에 실행. 자동 검증 수행
예: 코드 수정 후 자동으로 테스트/포맷팅

# 보안 폴더 접근 시 반드시 확인 요청
hook PreToolUse:
  "secrets/ 폴더 접근 시 → 무조건 나에게 물어볼 것"

신기능

2026년 새로 등장한 개념들

💰 태스크 예산 (Task Budgets)

에이전트가 사용할 최대 토큰량을 미리 설정하여 비용 폭주를 방지합니다. "이 작업은 10만 토큰 안에서 끝내"

👥 에이전트 팀 (Agent Teams)

하나의 작업을 여러 전문 에이전트에게 분산. 코드 리뷰어, 테스트 러너, 탐험가가 병렬로 협력합니다.

🧠 적응형 사고 (Adaptive Thinking)

작업 난이도에 따라 추론 시간을 스스로 조절. PM은 노력 수준(xhigh, max)을 설정하여 성능/비용을 관리합니다.
